Hi there!
HTML stands for Hypertext Markup Language

Markup is a form of text that can be interpreted in a certain format for design purposes. It includes references, which could be videos, links, etc.

Definitions

Markup - code that tells the browser what the text on a web page should look like

Hypertext - text that can have links or references

Nesting - the idea that some bits of code can go inside other bits of code

Syntax - the grammar rules of language



p is a block element, br is an inline element. The browser constructs invisible boxes around the elements.
Note that p does not need a closing tag, it is optional, the browser will figure out the closing tag on its own
The br is a void element, so unlike p, it has no contents

The div element:

A block level element
Div is short for division, you can divide page into different sections
div has invisible box around it, like p; and can also be formatted
p is specifically for text, div is generic and contains whatever

Lists:

ol and ul always need closing tags, li does not
ol is ordered list - with numbers
  1. dog
  2. cat
ul is unordered list - with bullets
  • dog
  • cat
li and p have optional closing tags, browser figures out where to close it because the browser knows there has to be a start tag when it gets to the start tag of a new element that cant be inside the last one. For example, when it gets to a new p element, it knows previous p element must be closed
Nested List Example

  • bad

Adding Images

src is source of image, where image is from
alt is text displayed if image is not found
img is a void element, an empty element, img is inline element
To create an image that is a link, put an img element between the open and close tags of an a element

Files and Relative URLs

a URL without a :// is always treated as a relative URL, cpu thinks its the name of a file
for a link to refer to a website, it has to be an absolute url with http
Attributes always go inside of the anchor, attributes include alt, src, href

Head and Body

head includes title, style, scripts, metadata search engines
body refers to everything else
if you dont include head and body tags, the browser will attempt to place them in your code itself
so head elements must appear first, body after
you cant have a head element such as title in the middle of your document

CSS Vocab

DOM-Document Object Model-tree structure with node for each HTML element, text, image, any other object
Selectors-part of css ruleset, indicate which html element the rule is for
Declaration blocks-other part of css ruleset, say how rule will modify elements indicated by selectors
CSS rules-composed of a selector followed by a declaration block
type-simplest kind of selector. Indicates the name of one type of HTML element such as em or li
class-an attribute that groups elements together that you want to have the same style
id - an attribute that must be unique to a single element
